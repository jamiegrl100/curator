Markdown# Installation Checklist (Preflight + Postflight)Use this checklist to avoid the most common install issues and to confirm your Curator Engine install is production-ready.---## Preflight (before installing)### Server basics- [ ] You have a Linux server/VPS with SSH access- [ ] Apache or Nginx is installed and running- [ ] PHP is installed (8.x recommended)- [ ] MySQL or MariaDB is installed and running### DNS / domain (if public)- [ ] Your domain points to the server IP- [ ] You can load a basic page from the domain### File locations- [ ] You have a dedicated folder for Curator (example: `/var/www/curator`)- [ ] Your web root points to Curator’s `public/` folder (not the project root)### Permissions / SELinux (RHEL/Alma/Rocky)- [ ] You know which user runs the web server (`apache` on Alma/RHEL, `www-data` on Debian/Ubuntu)- [ ] Writable directories (storage/uploads/logs/tmp) are owned or writable by that user- [ ] SELinux contexts are set for writable dirs (if SELinux is enforcing)### Database- [ ] Database created (example: `curator`)- [ ] DB user created (example: `curator_app@localhost`)- [ ] DB user has privileges on the Curator DB- [ ] You can connect with the credentials---## Install steps (minimum)- [ ] Download or clone Curator Engine- [ ] Copy config (`.env.example` → `.env`, or configure `config.php`)- [ ] Set:  - [ ] `APP_URL`  - [ ] DB host/name/user/pass- [ ] Install schema:  - [ ] Run installer script **or**  - [ ] Import `install/curator_schema.sql`  - [ ] Import seed file (if included)---## Postflight (after install)### Site loads- [ ] Home page loads without errors- [ ] Admin login works- [ ] Admin dashboard loads quickly### Core flows- [ ] Create a digital product- [ ] Upload at least one product image- [ ] Upload at least one product file- [ ] Product page renders correctly### Downloads- [ ] Purchase flow works (if payments configured)- [ ] “My Downloads” page shows purchased files- [ ] Token rules work:  - [ ] expiry (if enabled)  - [ ] max downloads  - [ ] revoke/regenerate### Email (recommended)- [ ] SMTP configured (optional but strongly recommended)- [ ] Test email succeeds- [ ] Transactional email queue processes successfully### Logs & diagnostics- [ ] Error logs are accessible (Apache/Nginx + PHP)- [ ] Curator app logs write to `storage/logs/` (if implemented)- [ ] No repeated warnings/errors on normal page loads---## Production hardening (recommended)### HTTPS- [ ] HTTPS enabled (Let’s Encrypt recommended)- [ ] HTTP redirects to HTTPS- [ ] No mixed-content warnings### Security- [ ] Admin passwords are strong- [ ] `.env` is not accessible from the web- [ ] `storage/` is not web-accessible- [ ] CSRF protection enabled on forms- [ ] Upload restrictions (type/size) enforced### Backups- [ ] Database backups scheduled- [ ] Storage backups scheduled (uploads/files)### Performance- [ ] PHP OPcache enabled (recommended)- [ ] Large catalog settings tested (if applicable)- [ ] Search indexes exist and queries are fast---## Troubleshooting quick hits### “500 error”- Check Apache/Nginx error logs- Check PHP-FPM logs (if used)- Confirm file permissions + SELinux contexts### “Database connection failed”- Verify DB host/user/pass- Confirm DB user privileges- Confirm correct DB selected### “Downloads denied”- Confirm order is paid- Confirm token belongs to user- Check expiry/max downloads/revoked status---## Next steps- [Installation](./INSTALLATION.md)- [Quick Start](./QUICK_START.md)- [FAQ](./FAQ.md)Copy
