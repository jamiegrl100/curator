Markdown# Security PolicyCurator Engine takes security seriously. If you believe you’ve found a vulnerability, please report it responsibly.## Supported VersionsSecurity fixes are applied to the latest stable release. If you are running an older version, upgrade to the latest release before reporting.## Reporting a Vulnerability**Please do not open a public GitHub issue for security vulnerabilities.**Instead, report privately:- Email: **security@snippetsupply.com** (replace with your preferred address)- Subject: `Curator Security Report - <short summary>`Include:- A clear description of the issue- Steps to reproduce (proof-of-concept if available)- Impact (what an attacker can do)- Affected version(s)- Any logs or screenshots (sanitize secrets)If you do not have a security email set up yet, you can temporarily use your main support email, but a dedicated security inbox is recommended.## What to ExpectAfter you report:1. We will acknowledge receipt within **3–7 days**.2. We will investigate and may request additional details.3. We will work on a fix and coordinate a release.4. If appropriate, we will credit you (optional) in release notes.## ScopeExamples of in-scope issues:- Remote code execution (RCE)- Authentication bypass- SQL injection- Stored or reflected XSS- CSRF that leads to account compromise- Arbitrary file read/write- Plugin pipeline bypass (publishing unreviewed artifacts)- Zip Slip / path traversal in ZIP extraction- Broken access control (downloading other users’ files)Out of scope (generally):- Social engineering- Issues requiring unrealistic user interaction- Denial of service from very large inputs unless it’s trivial and practical## Security Best Practices (for users)- Keep Curator updated to the latest release- Use HTTPS- Use strong admin passwords- Do not expose admin routes publicly without protection- Store secrets securely (.env, encrypted settings)- Limit file permissions (uploads/logs writable only where required)## Plugin/Theme SecurityCurator Marketplace reviews plugins and themes for security and policy compliance. Do not install plugins/themes from untrusted sources.Plugins should:- Avoid obfuscation- Avoid remote code execution or “phone home” auto-updaters- Declare capabilities explicitly- Use prepared statements for DB access- Validate and sanitize all user input## DisclosureWe prefer coordinated disclosure to protect users. Please give us time to patch before publishing details publicly.Copy
