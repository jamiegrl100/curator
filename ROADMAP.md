Markdown# RoadmapCurator Engine is a **search-first ecommerce core** designed to be extended through **themes** and **plugins**. This roadmap shows the direction of the project and what we’re prioritizing next.> This roadmap is not a contract. Priorities may shift based on stability, security, and community needs.---## Current focus### Core stability & performance- Improve install/upgrade reliability- Reduce friction in admin flows- Performance tuning for large catalogs (100k+)### Ecosystem foundations- Developer docs and examples- Safer plugin review pipeline- Marketplace UX improvements---## Near-term (next releases)### Engine- Better install wizard / diagnostics- More admin export options (orders/products/logs)- Improved logging and error reporting- SEO improvements (sitemaps, metadata coverage tools)### Themes- More starter themes (light/dark)- Theme preview and screenshot improvements- Theme best-practice guidance### Plugins- More core hook coverage + a stable hooks reference- Plugin scaffolding/generator tooling- More robust safe mode and plugin error isolation---## Mid-term### Commerce- Coupons/discounts (likely as a plugin)- Abandoned cart workflows (plugin)- More payment gateways (plugin packs)### Marketplace- Better discovery: featured, trending, verified filters- Ratings/reviews (planned as a plugin)- Automated compatibility testing (future)### Enterprise-ready features (mostly plugins)- Audit logs- Team roles and permissions- Advanced analytics/reporting packs---## Long-term### Physical goods (plugin)- Shipping zones/methods- Inventory tools- Tax enhancements and integrations### Multi-vendor marketplace (enterprise plugin)- Vendor onboarding + payouts (Stripe Connect Express)- Commission rules- Vendor dashboards---## What you can help with- Bugs and fixes- Docs improvements- Example plugins/themes- Performance profiling and scaling reports- Security review and hardening suggestions---## VersioningCurator aims to follow semantic versioning where possible:-
